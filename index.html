📖 HR Suite Pro — Help & Guide

Complete guide for all modules · Version 3.0 · Innova Code

First Time Setup

1
Add Employees
Go to Employees → click + Add Employee. Fill in name, department, basic salary, and join date. These are used across all modules.
2
Configure Work Schedule
Go to Settings → set work start/end time, grace period, and OT threshold. This drives attendance status calculations.
3
Connect Device (optional)
Go to IVMS / Device → enter your fingerprint/face device IP. Use Pull Device in Daily View to auto-import attendance.
4
Run Payroll
Go to Run Payroll → select month → click Generate. Review, adjust OT/deductions, then Finalize.
5
Enable Cloud Sync (optional)
Go to Settings → paste Firebase config JSON → click Save & Connect. All data syncs to cloud.

Data Storage

💾
All data is saved to localStorage in your browser. It persists across sessions on the same browser/device. Connect Firebase for multi-device sync and backup.
⚠
Clearing browser data / cache will delete all locally stored records. Firebase sync is strongly recommended for production use.

User Roles

👑 Admin
Full access to all modules including Users management and sensitive payroll data.
👷 Staff
Access limited to modules permitted by Admin. Configured per user in Settings → Users.

Attendance

📋
Daily View
Mark attendance per employee. Supports Present, Late, Absent, Half Day, Holiday. Inline time entry.
📅
Monthly Grid
Full month view — colour-coded attendance grid per employee. Export to CSV.
📥
CSV Import
Bulk import attendance from CSV/Excel. Download template, fill, upload. Duplicate detection included.
Daily View — How to use:
• Quick Mark All bar — mark all employees P/L/A/H at once.
• Pull Device — auto-pull check-in/out from IVMS fingerprint device.
• Manual Entry — add or edit individual records.
• Status colours: 🟢 Present · 🟡 Late · 🔴 Absent · 🔵 Holiday · 🟠 Half Day
Monthly Grid: Select month → grid renders all employees × days. Click any cell to edit that day's record. Export button exports entire month to CSV.
CSV Import: Required columns: emp_id att_date status. Optional: check_in check_out. Dates: YYYY-MM-DD format.

HR Management

👥
Employees
Full employee registry with profile cards, photos, employment details, and history.
🏖
Leave
Apply and approve leave requests. Tracks Annual (14 days), Casual (7), Sick (7), Maternity (84) and No-Pay leave balances.
🕐
OT Tracker
Record overtime hours with 1.5x / 2x rates. Approval workflow: Pending → Approved → Paid.
📅
Shift Roster
Assign shifts to employees. Weekly grid view. Manage shift templates (Morning, Day, Evening, Night).
💸
Advances & Loans
Issue salary advances or loans. Auto-calculates monthly deductions and tracks outstanding balance.
📋
Disciplinary
Record verbal/written/final warnings, suspensions, and terminations with full incident details.
📄
Documents
Store employee document records (NIC, Passport, Contract etc.). Attach real files — saved to a folder you select. Expiry alerts included.
🎓
Training
Log training programs and attendees. Track completion status and certificate numbers.
🏦
Gratuity
Calculates Sri Lanka statutory gratuity for eligible employees (5+ years). Formula: Basic ÷ 2 × Years.
📢
Notice Board
Post company announcements with Urgent / Info / Normal priority. Pin important notices to top. Set expiry dates.
Employees: Key fields for payroll: Basic Salary, Department, Status (Active/Inactive/Probation), EPF eligibility. Join Date is used for Gratuity calculations. Employee No. is used for document file naming.
Leave Balances are calculated from join date. No-Pay leave automatically triggers a salary deduction when payroll is run for that month.
OT Rates: Normal Day = 1.5× hourly rate · Holiday/Sunday/Poya = 2× · Hourly rate = Basic ÷ 26 days ÷ 8 hours.
Shift Templates: Go to ⚙ Shift Templates to create reusable shifts. When assigning, select a template to auto-fill times, or enter custom times.
Loan Deductions are not yet automatically applied to payroll — manually enter as a deduction in the payroll run for that employee.
Document Files: Settings → Select Documents Folder first. Files are saved as EMPNO_Category_Name.ext in category sub-folders. Reload requires re-selecting folder (browser security).
Bulk Add: Click "+ All Employees" in the Attendees tab to add the entire workforce to a training program at once.
Sri Lanka Gratuity Act: Minimum 14 days wages per completed year of service, payable after 5 years of continuous employment. This module shows estimated values — consult HR/Legal for final figures.
Pinned Notices appear at the top of the board. Expired notices (past expiry date) are automatically hidden from the board.

Running Payroll

1
Select Month
Choose the payroll month. The system will use attendance data from that month.
2
Generate
Click Generate — basic salary, allowances, EPF/ETF, and PAYE are auto-calculated for all active employees.
3
Adjust OT / Deductions
Click the OT button per employee to add overtime hours, extra allowances, or other deductions.
4
Finalize
Click Finalize to lock the payroll. This saves it and makes it available for payslips and EPF/ETF reports.

Sri Lanka Statutory Compliance

EPF — Employee
8% of basic salary (deducted from employee)
EPF — Employer
12% of basic salary (employer contribution)
ETF — Employer
3% of basic salary (employer contribution)
PAYE
IRD 2024/25 tax slabs · Tax-free threshold: LKR 1,800,000/year
Gratuity
Eligible after 5 years · Basic ÷ 2 × Years of service

EPF / ETF Payments

📅
EPF contributions must be remitted to the Labour Department by the last working day of the following month. ETF to the ETF Board by the last day of the following month.
📋
Use the EPF/ETF module to generate C-Form / R-Form summaries per month and record payment details (cheque number, bank, payment date).

Payslips

After finalizing payroll, go to Payslips → select employee and month → click Print / PDF. Payslips include earnings, deductions, EPF/ETF breakdown, and net pay.

Setting Up Firebase

1
Create Firebase Project
Go to console.firebase.google.com → New Project.
2
Add Web App
Project Settings → Your Apps → Add Web App. Copy the config object (apiKey, projectId, etc.).
3
Enable Firestore
Build → Firestore Database → Create database → Start in test mode (or set security rules).
4
Paste Config
Settings → Firebase Config → paste the JSON → Save & Connect → Test Connection.

Firestore Security Rules

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
🔒
The above rules allow open access — suitable for internal networks only. For public deployments, add authentication rules.

Pro Tips

🎨
Themes: Click the 🎨 icon in the topbar to switch between Navy Blue, Forest Green, Warm Amber, Deep Purple, OLED Black, Slate Grey, and Light Mode.
📤
Exports: Most modules have a CSV export button. Use it for monthly payroll reports, attendance, OT, and disciplinary records.
🔄
Sync Button: The cloud icon in the topbar shows sync status. Click it to manually trigger a full Firebase sync (re-downloads all data).
📁
File Folders: Photo folder and Documents folder must be re-selected after each browser reload. This is a browser security restriction, not a bug.
👤
User Menu: Click your name/avatar in the topbar to see your role and log out.
📌
Notice Board: Pin urgent notices so they always appear at the top. Set an expiry date so old notices are automatically hidden.
